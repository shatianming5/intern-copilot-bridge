# J_0030 Slash esc clears pending input journey

<!-- METADATA:TYPE=J,ID=J_0030,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=8 -->

## Concise proposal

Prompt: make Codex wait at a pending input/question state.

Do: send `/esc`.

Prompt: send a new unrelated instruction.

Look for: the pending input is cleared, the new instruction is handled cleanly,
and no stale pending answer leaks into the next turn.

## Goal

Validate the user-facing `/esc` recovery path:

- a pending input/question state exists;
- `/esc` clears that state through the real product path;
- the session remains usable;
- the next prompt is not polluted by the cleared pending input;
- pending cards are closed or marked stale.

## Why J

F can test slash routing. This case proves a real user can recover a stuck
conversation and continue naturally.

## Cost profile

Paid/agent. The case induces a real pending state and sends real Feishu slash
and follow-up messages.

## Case resources

- namespace: `ci_j_0030`
- intern: `intern_ci_j_0030_codex_<run_id>`
- stale token: `CI_J0030_STALE_<run_id>`
- fresh token: `CI_J0030_FRESH_<run_id>`
- report prefix: `j_0030_esc_clears_pending_input`

## Scenario

| # | Type | Operation | Expected |
|---|------|-----------|----------|
| 1 | action | Clean only `ci_j_0030` resources. | Previous intern, group, pending questions, and reports are removed or isolated. |
| 2 | action | Create Codex intern, session, and Feishu group. | Group is green. |
| 3 | action | Prompt Codex to ask one pending confirmation containing stale token. | A pending question/card or pending input state appears. |
| 4 | action | Send `/esc` through the real Feishu slash path. | Slash command is delivered to the target intern/session. |
| 5 | wait/assert | Wait for escape effect. | Pending input/question is cleared or card is marked stale/cancelled. |
| 6 | action | Prompt: "忽略刚才的问题，回复 fresh token。" | New prompt is normal text. |
| 7 | wait/assert | Wait for reply. | Reply contains fresh token and does not treat stale token as an answered choice. |
| 8 | assert | Check report. | Report links pending state, slash message, clear evidence, new prompt/reply, and stale-card status. |

## Key assertions

- `/esc` clears pending state without killing the session.
- Next prompt is fresh and unpolluted.
- Old card/input cannot later answer into the new turn.

## Implementation prerequisites

- Pending-state inducer.
- Real slash driver for `/esc`.
- Pending question/card stale-state reader.

## Failure classification

- `/esc` does not clear pending input: product bug.
- Session becomes unusable after `/esc`: product bug.
- Old pending answer leaks into next prompt: product bug.
- CI cannot create a stable pending state: CI helper gap.
