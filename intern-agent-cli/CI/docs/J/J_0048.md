## Concise proposal

Prompt: `分配任务 <task_id>。`

Prompt: `继续。`

Prompt: `review 里 focused test pytest tests/test_ci_j_0048_review_gate.py::test_expected_token 失败了：期望输出 CI_J0048_FIXED_<run_id>，当前输出 CI_J0048_DRAFT_<run_id>。请修在同一个 MR/PR 上。`

Look for: the intern fixes the failure and pushes a new revision to the same
change request, without opening a duplicate or merging.

## Goal

Validate review-feedback handling for failed checks:

- assignment and initial work start from real Feishu prompts;
- supervisor reports a concrete focused test failure;
- intern diagnoses and fixes the same branch;
- same MR/PR receives a new revision;
- final reply includes rerun evidence;
- merge still waits for approval.

## Why J

This is a real supervisor loop: "tests failed; update your PR." It requires
agent reasoning and provider revision evidence.
