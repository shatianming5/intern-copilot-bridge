# J_0095 Coordinator workspace task handoff journey

<!-- METADATA:TYPE=J,ID=J_0095,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=9 -->

## Concise proposal

Prompt: ask a coordinator to create and hand off a small workspace task to a
team lead.

Look for: one scoped task is created, team lead sees it, and coordinator reports
handoff evidence back to the user.

## Goal

Validate coordinator-to-team-lead task handoff:

- coordinator receives a natural work request;
- it creates or selects one scoped task;
- team lead receives the handoff;
- user gets task id, assignee, and next-step evidence.

## Why J

Task metadata commands are F. This case proves the supervisor can ask a
coordinator to organize work and receive a coherent handoff summary.

- Handoff is explicit and scoped.
- Team lead receives the same task the coordinator reports.

## Implementation prerequisites

- Coordinator/team lead fixture.
- Task creation and notification evidence.

## Failure classification
