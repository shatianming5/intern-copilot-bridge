# J_0074 Long no-collapse follow-up ordering journey

## Scope note

Keep this case only as no-collapse/message-splitting product coverage. The
follow-up prompt should reference an explicit late token so the assertion is
timeline order and context continuity, not broad long-answer quality.

## Concise proposal

Do: enable no-collapse for a long answer.

Prompt: ask for a long structured report, then ask a follow-up referencing item
near the end.

Look for: long answer continuation messages stay ordered and the follow-up can
reference late content.

## Goal

Validate long-output continuity:

- no-collapse mode is active;
- intern sends multiple ordered continuation messages;
- follow-up refers to a late section;
- intern can answer based on late section without losing order.

## Why J

Message splitting/order is user-visible only in a real Feishu conversation.
