# J_0045 Claude skill visibility and use journey

<!-- METADATA:TYPE=J,ID=J_0045,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=5 -->

## Concise proposal

Prompt: ask Claude to use the enabled repo skill on a tiny input.

Look for: Claude replies with the skill's required token, fixed format, and the
processed input.

Do: disable the repo skill while keeping the same intern/session scope.

Prompt: ask Claude to use that repo skill again.

Look for: Claude must not claim it used the disabled repo skill. It may say the
skill is unavailable or answer without the skill token.

Do: enable the same skill as personal scope.

Prompt: ask Claude to use the personal skill on another tiny input.

Look for: Claude again follows the skill contract, proving real agent-visible
skill availability after scope change.

## Goal

Validate that a Claude intern can actually see and use enabled skills in a real
user journey:

- a case-scoped skill defines a visible token and output format;
- the skill is enabled for the Claude intern;
- the user asks Claude to use the skill on a small input;
- Claude's reply follows the skill contract;
- disabling the repo skill changes what Claude can claim/use;
- enabling the skill in personal scope restores real agent-visible use.

## Why J

`F_0045` validates Claude skill farm paths, `.claude/skills` projection,
repo/personal enable/disable state, TreeView projection, and group mode parity
from files, registry, CLI/API, and source evidence. This case is J-scoped
because it asks Claude to perform a task using the skill and validates the
agent's reply. That real agent behavior cannot be proven by file or registry
checks alone.

## Cost profile

Paid/agent. The case starts a real Claude session, sends natural prompts, waits
for Claude replies, and records only redacted provider/token/runtime evidence.

## Required environment

- Backend: `claude`.
- A lead-prepared local or throwaway workspace suitable for Claude skill tests.
- Claude provider token/policy material may be loaded only through the approved
  policy alias used by F Claude cases. Secret values must never be written to
  docs, stdout, reports, or retained artifacts.
- The workspace mode must be fixed at add time; this case must not add, delete,
  migrate, or switch workspace mode.
- F-level skill farm/source/TreeView contracts should already be covered by
  `F_0045`; this J focuses on real Claude behavior after those precondition actions.

## Skill fixture

The case-scoped `SKILL.md` must be small and deterministic. Example contract:

```text
# CI J0045 Echo Formatter

When the user asks you to use this skill, reply in exactly this shape:

CI_J0045_SKILL_USED
input=<the user's short input>
result=<the input uppercased>

Do not add any other lines.
```

The runner may vary the token by run id if needed, but the token must be
recorded in the report and must not include secrets.

## Key assertions

- Claude type does not drift in status/list, session map, group registry, or
  report evidence.
- Enabled repo skill use produces the exact skill token and expected output
  format.
- Disabled repo skill cannot still appear as a successful repo-skill use.
- Personal skill enablement restores agent-visible skill use.
- Claude skill projection uses `.claude/skills`, not `.agents/skills`, when the
  report references farm evidence.
- Token, policy, and runtime evidence are redacted.
- The case never adds, deletes, migrates, or switches the workspace.

## Failure classification

- Claude cannot start or type drifts to another backend: product bug.
- Skill farm/projection uses `.agents/skills` for Claude: product bug, already
  aligned with the F_0045 failure class.
- Enabled skill is not visible to Claude or Claude cannot follow the skill
  contract: product bug or model behavior finding, classified by evidence.
- Disabled repo skill is still claimed as successfully used with the skill
  token: product bug.
- Personal skill enablement is not visible to Claude: product bug.
- Claude prompt/reply or skill-token extraction cannot be observed: CI helper
  gap.
- Secret provider token or policy material appears in stdout/report/artifacts:
  CI bug.
- Workspace cleanup, deletion, migration, or mode switching occurs: CI bug.
