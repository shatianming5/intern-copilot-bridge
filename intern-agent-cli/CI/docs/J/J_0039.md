# J_0039 Stop while pending question journey

<!-- METADATA:TYPE=J,ID=J_0039,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=8 -->

## Concise proposal

Prompt: make the intern ask a pending question.

Do: send `/stop` before answering.

Prompt: send a fresh instruction.

Look for: the pending question is cancelled/stale, the session accepts the fresh
instruction, and late answers to the old card do not apply.

## Goal

Validate `/stop` recovery from a pending question:

- pending card exists;
- `/stop` cancels or stales the pending state;
- session remains usable;
- late card answer is rejected;
- new prompt starts cleanly.

## Why J

`/stop` is already covered for running turns in `J_0008`; this variant covers
the common stuck-decision path.

## Key assertions

- `/stop` clears pending question state.
- Late answers cannot affect future turns.

## Implementation prerequisites

- Real `/stop` slash driver.
- Pending card stale/cancel evidence.

## Failure classification

- Stop leaves pending card active: product bug.
- Late answer mutates new turn: product bug.
