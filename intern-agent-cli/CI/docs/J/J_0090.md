# J_0090 Feishu artifact delivery bundle journey

<!-- METADATA:TYPE=J,ID=J_0090,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=12 -->

## Concise proposal

Prompt: ask the intern to send a small image, then a small file, then an
oversized report in separate natural turns.

Look for: successful artifacts have real Feishu image/file evidence and rich
content message boundaries; oversized delivery gets a clear failure/fallback
notice instead of silent loss or a fake "sent" claim.

## Goal

Validate outgoing artifact delivery as one user journey bundle:

- the intern naturally uses the Feishu messaging path to send a small image;
- the intern naturally sends a small report file in a later prompt;
- text before and after each rich content send stays ordered in Feishu;
- oversized or unsupported outgoing content surfaces a truthful failure notice;
- fallback summary/path is offered when possible.

## Why J

Transport limits and upload API return codes can be tested in F. This case is J
because the supervisor asks a real intern to send artifacts and expects honest
user-visible delivery results across success and failure paths.

## Key assertions

- Rich content sends come from the intern's natural tool/skill use, not direct
  runner injection into Feishu APIs.
- Text before image/file is not edited after the rich content message is sent.
- Text after image/file uses a newer Feishu message id.
- Silent oversized delivery loss is a failure.
- Failure notice is actionable and does not leak secrets or raw oversized
  content.

## Implementation prerequisites

- Built-in `feishu-messaging` or equivalent approved Feishu artifact send path.
- Tiny image/file artifact fixture.
- Oversized artifact fixture with redacted/no-secret content.
- Feishu send-result and timeline evidence that can distinguish message ids,
  image/file message types, and failures.

## Failure classification

- Fake sent claim after send failure: product bug.
- Silent dropped output: product bug.
- Text after rich content updates the old pre-rich message: product bug.
- CI sends artifacts directly through daemon/API instead of agent tool use: CI
  bug.
