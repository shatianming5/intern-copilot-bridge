# J_0040 Idle session restart follow-up journey

<!-- METADATA:TYPE=J,ID=J_0040,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=8 -->

## Concise proposal

Prompt: send a small message to a live idle Codex intern and capture session
evidence.

Do: restart the idle session through the product session restart path.

Prompt: send a follow-up in the same Feishu group.

Look for: the group binding remains valid and the restarted session answers
without creating a duplicate intern/group.

## Goal

Validate idle restart continuity from the user's view:

- an idle session has a working Feishu group;
- product restart path restarts the session;
- group/chat binding survives;
- follow-up prompt reaches the restarted session;
- no duplicate intern/group/session registry entry is created.

## Why J

F can validate restart command output. This case proves the user can keep using
the same group after restart.

- Same group remains usable after restart.
- Restart does not duplicate intern/group records.
