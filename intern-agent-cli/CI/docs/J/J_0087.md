# J_0087 Blocked goal resume continuation journey

<!-- METADATA:TYPE=J,ID=J_0087,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=12 -->

## Concise proposal

Prompt: set a goal that cannot complete until the supervisor supplies a missing
case value, then resume the same goal after the value is provided.

Look for: blocked status is honest, resume uses the same goal lifecycle, and no
duplicate goal is created.

## Goal

Validate blocked-goal recovery:

- agent reports a genuine blocker with evidence;
- goal remains blocked rather than silently completed;
- user supplies the missing input and resumes;
- continuation belongs to the same goal.

## Why J

This is a multi-turn supervisor journey around goal lifecycle, not just daemon
goal state mutation.

## Scenario

| # | Type | Operation | Expected |
|---|------|-----------|----------|
| 1 | action | Prompt `/goal 继续处理 ci_j_0087 的耗时目标：你必须等主管提供 CI_J0087_MISSING_VALUE_<run_id> 后才能完成；缺失值未提供前，请把目标保持为 blocked，并报告 CI_J0087_BLOCKED_<run_id>。` | Goal becomes active and cannot complete immediately. |
| 2 | wait/assert | Let the agent reach the missing value. | Agent reports blocked with concrete missing-value evidence and includes blocker token. |
| 3 | action | Prompt `缺失值是 CI_J0087_MISSING_VALUE_<run_id>=CI_J0087_RESUME_<run_id>，继续原 goal，不要新建目标。` | Agent resumes the existing blocked goal. |
| 4 | wait/assert | Wait for final reply. | Reply uses the resume token and resolves the original goal. |
| 5 | assert | Check goal lifecycle evidence. | One goal lifecycle is observed: active -> blocked -> resumed -> completed; no duplicate `/goal` replace/clear/new-goal path is used. |

## Key assertions

- Blocker evidence is concrete and tied to the missing value marker.
- Resume does not start an unrelated goal or require Replace confirmation.
- Final answer and `/goal status` agree that the original goal is resolved.

## Implementation prerequisites

- Goal lifecycle event reader.
- Controlled missing-input fixture.
- Transcript/timeline correlation between the blocked report, resume prompt, and
  final answer.

## Failure classification

- Fake completion while blocked: model behavior finding.
- Resume creates duplicate active goals: product bug.
- Missing-value fixture completes before blocked state can be inspected: CI
  script bug.
