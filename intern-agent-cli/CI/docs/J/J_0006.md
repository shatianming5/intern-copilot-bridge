# J_0006 Codeup metadata_branch to repo_dotdir migration journey

<!-- METADATA:TYPE=J,ID=J_0006,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=6 -->

## Concise proposal

Do: start from an exclusive Codeup workspace in `metadata_branch` mode with real
task/status/history/knowledge/skill metadata.

Do: run `internctl workspace migrate-mode --repo-url <repo> --target repo_dotdir`.

Look for: a Codeup migration MR opens against the repository default branch and
copies the real `.intern_workspace` tree out of the metadata branch.

Do: merge the migration MR, then re-add or reuse the workspace in `repo_dotdir`
mode.

Look for: task/status/history/knowledge/skill metadata resolves from the repo
`.intern_workspace` tree, Feishu routing still targets the same project/intern
scope, and no old in-place mode switch path was used.

## Goal

Validate the replacement user journey for migrating a Codeup workspace from
`metadata_branch` to `repo_dotdir` mode:

- the old relay workspace record is released for the case scope;
- `internctl workspace migrate-mode` creates a real Codeup MR;
- the MR copies the real metadata tree from the metadata branch back into the
  repository `.intern_workspace` directory on the default branch;
- the user/operator merges the MR;
- the workspace is re-added or reused in `repo_dotdir` mode;
- metadata and Feishu routing still work after migration.

## Why J

The old J_0006 in-place mode switch story is retired. This replacement is
J-scoped because it is an end-to-end operator journey across workspace records,
Codeup MR review/merge, workspace re-add/reuse, metadata resolution, and
user-visible intern routing. F cases can validate individual CLI guards and
migration fixtures; this J validates the full migration experience.

## Cost profile

Operator journey. It may not require paid agent output, but it does require a
real Codeup MR and a real workspace re-add/reuse path. It must run only in an
exclusive throwaway workspace and repository prepared for migration tests.

## Required environment

- Provider: `codeup`.
- Initial workspace mode: `metadata_branch`.
- Target workspace mode: `repo_dotdir`.
- A dedicated non-production Codeup repository with a configured metadata
  branch and a default branch that can receive the migration MR.
- Exclusive ownership of the test workspace and relay record for the duration
  of the case.
- No `local_only` workspace is involved.

## Key assertions

- Migration is explicit: release old record, create Codeup MR, merge MR, then
  re-add/reuse workspace.
- The migration MR copies real metadata branch content into repo
  `.intern_workspace`.
- Metadata resolves from the repo `.intern_workspace` tree after the workspace
  is re-added or reused.
- Feishu routing uses the new workspace record and remains scoped to the same
  project/intern identity.
- No old in-place workspace mode switch command or relay/daemon mode endpoint is
  used.

## Failure classification

- Migration MR is missing or targets the wrong branch: product bug.
- Migration MR loses metadata files or writes only markers: product bug.
- Re-add/reuse cannot resolve metadata from repo `.intern_workspace`: product
  bug.
- Feishu routing points at the released old record: product bug.
- Any old in-place mode switch path is used: CI bug.
- Cleanup mutates shared or lead-owned workspaces: CI bug.
