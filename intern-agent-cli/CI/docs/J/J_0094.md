## Scope note

Keep this case only as team metadata/no-mutation coverage. The valid assertion
is that a no-team intern does not create fake worker/task state; do not require
a particular generic refusal phrasing.

## Concise proposal

Prompt: ask an independent intern with no team to delegate work.

Look for: intern refuses or asks for a coordinator/team instead of inventing
workers or pretending delegation happened.

## Goal

Validate no-team delegation handling:

- independent intern has no team binding;
- user asks it to delegate a task to a team;
- intern reports that no team is configured or asks for explicit routing;
- no fake worker task/message is created.

## Why J

The tree/metadata flag is F. This case verifies the actual conversation when a
user asks for team behavior that is unavailable.
