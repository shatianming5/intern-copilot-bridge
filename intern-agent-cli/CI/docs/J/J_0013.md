# J_0013 Codex skill visibility and use journey

<!-- METADATA:TYPE=J,ID=J_0013,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=7 -->

## Concise proposal

Do: enable a deterministic repo skill for a Codex intern.

Prompt: ask Codex to use that skill on a tiny input.

Look for: Codex replies with the skill token, fixed format, and processed
input, proving the `.agents/skills` farm is actually agent-visible.

Do: disable the repo skill, then enable the same skill as personal.

Prompt: ask Codex to use it again after each scope change.

Look for: disabled repo scope is not claimed as successful use, and personal
scope restores the skill behavior for only the target intern.

## Goal

Validate that a Codex intern can actually see and use enabled skills in a real
user journey:

- a case-scoped skill defines a visible token and output format;
- repo-scope enablement is visible to Codex;
- repo-scope disablement changes what Codex can claim/use;
- personal-scope enablement restores visibility only for the target intern;
- all evidence comes from real Codex replies plus farm/config source evidence.

## Why J

`F_0030` validates Codex repo/personal skill enable/disable/promote and farm
sync from files, registry, CLI/API, and TreeView evidence. This case is J-scoped
because it asks Codex to perform a user task using the skill and validates the
agent's reply. File presence alone cannot prove model-visible skill use.

## Cost profile

Paid/agent. The case starts a real Codex session, sends natural prompts, and
waits for Codex replies.

## Required environment

- Backend: `codex`.
- A lead-prepared local or throwaway workspace suitable for skill tests.
- A case-scoped skill fixture with deterministic output.
- Skill sync on session start or an explicit sync/restart operation accepted by
  the product contract.

## Skill fixture

The case-scoped `SKILL.md` should be tiny and deterministic:

```text
# CI J0013 Echo Formatter

When the user asks you to use this skill, reply in exactly this shape:

CI_J0013_SKILL_USED
input=<the user's short input>
result=<the input reversed>

Do not add any other lines.
```

The runner may include a run-specific token if needed, but the token must be
recorded in the report and must not include secrets.

## Key assertions

- Enabled repo skill use produces the exact skill token and expected output.
- Disabled repo skill cannot still appear as successful repo-skill use.
- Personal skill enablement restores agent-visible use only for the target
  intern.
- Comparison intern remains unaffected by personal scope.
- Farm/config evidence and real replies agree.

## Implementation prerequisites

- A Codex prompt/reply helper that can extract deterministic skill token
  evidence.
- A skill fixture helper for repo and personal scope.
- A skill sync/restart helper if the live Codex session does not see farm
  updates without an accepted refresh operation.

## Failure classification

- Enabled skill is present in farm but not visible to Codex: product bug or
  model behavior finding, classified by transcript and farm evidence.
- Disabled repo skill still emits the skill token as successful use: product
  bug.
- Personal skill leaks to comparison intern: product bug.
- Skill token extraction is unreliable: CI helper gap.
