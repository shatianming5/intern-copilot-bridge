# J_0044 Claude group mode live-turn journey

<!-- METADATA:TYPE=J,ID=J_0044,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=8 -->

## Concise proposal

Do: configure a Claude intern group for trigger-all and at-only.

Prompt: send normal and addressed messages.

Look for: trigger modes affect real Claude turns the same way they affect user
expectations; detail mode changes visible timeline without hiding final answer.

## Goal

Validate Claude-specific group mode behavior:

- `trigger_mode=all` wakes Claude on normal messages;
- `trigger_mode=at_only` ignores non-addressed chatter;
- addressed message wakes Claude in at-only mode;
- summary/full detail modes affect visible tool/timeline detail;
- group config evidence and Claude reply evidence agree.

## Why J

`F_0045` can validate Claude group mode config parity. This case checks whether
the modes affect real Claude conversations.

## Key assertions

- Claude group mode affects real live turns.
- At-only suppresses normal chatter but not addressed prompts.
- Detail mode does not hide final answer.

## Implementation prerequisites

- Claude prompt/reply helper.
- Addressed-message driver.
- Bounded no-turn and timeline classifier.

## Failure classification

- At-only wakes on normal chatter: product bug.
- Addressed message does not wake Claude: product bug or driver gap.
- Summary/full hides final answer or has no observable contract: product bug or product gap.
