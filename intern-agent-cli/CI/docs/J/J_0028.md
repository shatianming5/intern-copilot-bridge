# J_0028 Goal-send permission boundary journey

<!-- METADATA:TYPE=J,ID=J_0028,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=8 -->

## Concise proposal

Prompt: ask an unauthorized independent intern to set another intern's pressing
goal.

Look for: the action is refused with a visible permission reason.

Prompt: ask an authorized coordinator to set and clear the same worker goal.

Look for: authorized path works and affects only the intended worker.

## Goal

Validate real user-facing `goal-send` permission boundaries:

- unauthorized independent intern cannot set/clear another intern's goal;
- authorized coordinator can set a worker goal;
- worker sees the active goal;
- authorized clear removes only that goal;
- all evidence comes from real agent skill use and goal state.

## Why J

F can validate the goal API contract. This case proves agents using the
builtin `goal-send` skill surface permission failures and successes correctly.

## Cost profile

Paid/agent. The case starts independent, coordinator, and worker Codex sessions.

## Key assertions

- Unauthorized goal-send is refused and does not mutate worker state.
- Authorized coordinator path works.
- Clear is scoped to the case goal.

## Implementation prerequisites

- Role/team fixture helper.
- Builtin skill-use evidence.
- Worker goal snapshot reader.

## Failure classification

- Unauthorized sender mutates worker goal: product bug.
- Authorized coordinator cannot set goal: product bug or role fixture gap.
- Clear removes unrelated goals: product bug.
- Agent refuses to use available skill despite clear prompt: model behavior finding.
