## Scope note

Keep this case only as detail-mode rendering/product coverage. It may assert
that required outcome tokens, validation, and blocker classification survive
summary rendering; it should not become a generic writing-quality test.

## Concise proposal

Prompt: run a tool-heavy turn in summary detail mode.

Look for: Feishu summary is readable, preserves the important outcome, and
does not collapse paragraphs into an unusable wall of text.

## Goal

Validate summary detail readability:

- detail mode is set by a precondition;
- agent runs a tool-heavy but harmless task;
- Feishu output uses summary rendering;
- summary preserves outcome, validation, and blocker classification.

## Why J

Mode persistence and render flags are F. This case validates the human-visible
summary after a real agent turn.
