# J_0012 Builtin skills compaction recovery journey

<!-- METADATA:TYPE=J,ID=J_0012,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=6 -->

## Concise proposal

Prompt: ask a fresh Codex intern with no repo/personal skills to send a small
artifact to its Feishu group.

Look for: the intern discovers and uses the built-in `feishu-messaging` skill,
not a prompt-tail hardcoded path.

Do: force a compact/resume or equivalent context-shortening boundary.

Prompt: ask the same intern to send a short peer message to a receiver intern,
then have a coordinator set and clear a worker goal.

Look for: `peer-send` and `goal-send` remain discoverable after compaction, the
receiver/worker sees the effects, and no repo/personal skill config was needed.

## Goal

Validate that default builtin collaboration skills are agent-visible in the
real user journey, including after context compaction:

- `feishu-messaging` can be used by a fresh Codex intern without repo/personal
  skill config;
- after compaction/resume, `peer-send` is still discoverable and usable;
- a coordinator-scoped intern can use `goal-send` to set and clear a worker
  pressing goal;
- the journey does not depend on old prompt-tail references to builtin docs.

## Why J

F can prove that builtin skills are copied into the farm, hidden from editable
TreeView groups, and protected from CLI mutation. This case is J-scoped because
it asks real agents to use those builtin skills before and after compaction and
validates the visible effects.

## Cost profile

Paid/agent. The case starts real Codex sessions, drives natural prompts, may
force compaction/resume, and waits for skill-driven product effects.

## Required environment

- Backend: `codex`.
- A sender intern, receiver intern, coordinator intern, and worker intern in a
  case-scoped workspace.
- Fresh repo/personal skill config: no user-managed skill is needed for the
  builtin names.
- Builtin skill sync on session start.
- A compaction/resume helper or equivalent product operation that shortens the
  model-visible conversation while keeping the same intern session usable.

## Key assertions

- A fresh intern can use builtin skills without repo/personal skill config.
- Builtin skills remain usable after compaction/resume.
- `peer-send` respects project/intern scoping and receiver evidence is visible.
- `goal-send` respects role/permission boundaries and affects only the case
  worker goal.
- The journey fails if the only evidence is farm files with no real agent use.

## Implementation prerequisites

- A compaction/resume helper or accepted equivalent source evidence.
- A way to observe model-visible skill use without leaking private prompts or
  relying on brittle exact transcript text.
- Receiver/worker evidence helpers for peer-send and goal-send effects.
- Role fixture for coordinator-permitted goal-send usage.

## Failure classification

- Builtins are present in farm but not discoverable by the agent: product bug
  or model behavior finding, classified by transcript/source evidence.
- Builtins disappear after compaction/resume: product bug.
- Peer-send targets the wrong project/intern or is not visible to receiver:
  product bug.
- Goal-send bypasses permission boundaries or clears unrelated goals: product
  bug.
- The runner cannot perform or observe compaction safely: CI helper gap.
