# J_0067 Worker clarification routed through coordinator journey

<!-- METADATA:TYPE=J,ID=J_0067,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=9 -->

## Concise proposal

Prompt: coordinator delegates an ambiguous worker task.

Look for: worker asks for clarification, coordinator routes the question to the
user or answers from context, and worker resumes with the clarified scope.

## Goal

Validate clarification in team workflows:

- worker receives ambiguous delegated task;
- worker does not guess;
- clarification reaches coordinator/user path;
- clarified answer returns to worker;
- worker completes the scoped task.

## Why J

This is a multi-agent conversation flow, not only role metadata.

## Key assertions

- Worker clarification is visible and routed.
- Worker does not act on ambiguous scope before clarification.

## Implementation prerequisites

- Team message routing evidence.
- Clarification card/text helper.

## Failure classification

- Worker guesses without clarification: model behavior finding.
- Clarification never reaches worker: product bug.
