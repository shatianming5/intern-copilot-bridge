# J_0038 Pending question answered by normal text journey

<!-- METADATA:TYPE=J,ID=J_0038,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=8 -->

## Concise proposal

Prompt: make the intern ask a free-text question.

Do: answer as a normal Feishu text message instead of clicking the card.

Look for: the pending question consumes the text answer and the intern
continues; the text is not also treated as a separate new task prompt.

## Goal

Validate normal-text answers to pending questions:

- pending question exists;
- user sends normal text while it is pending;
- text is consumed as the answer;
- no duplicate independent turn is started from the same text;
- intern continues with that answer.

## Why J

The user may reply naturally rather than use the card UI. This requires a live
conversation and pending question state.

## Failure classification
