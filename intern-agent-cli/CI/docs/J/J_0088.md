## Concise proposal

Prompt: run a case-scoped long read-only journey until Codex compacts, then
send a token follow-up.

Look for: compaction summary is internal recovery context, not a confusing
assistant answer in Feishu, and the next user prompt still works.

## Goal

Validate user-visible compaction behavior:

- long transcript triggers compaction/resume;
- internal summary does not appear as a standalone answer to the user;
- follow-up prompt is answered normally;
- task and newest instruction remain intact.

## Why J

spurious recovery text and can continue the conversation.
