# J_0033 Codex exit resume same-session journey

<!-- METADATA:TYPE=J,ID=J_0033,STATUS=Implemented,OWNER=intern_ci_lead,SESSION=119 -->

## Goal

Validate the user-visible Codex resume journey after a real Codex turn:

- start a Codex intern session in a case-scoped Feishu group;
- send a minimal `hi` prompt and wait for the Codex reply;
- capture the durable Codex session UUID through product/session evidence;
- send `/exit`, capture the printed `Resume this intern` command, execute it in tmux, and require the same UUID;
- run the GUI-equivalent restart command and require `restarted via resume <uuid>` with the same UUID.

## Why J

F-level restart cases do not send natural-language prompts. Without a real Codex
turn, there may be no durable Codex session UUID, so a no-prompt restart may
legitimately start fresh. This case is J-scoped because it sends a user prompt
and waits for a live Codex reply before asserting same-session resume.
