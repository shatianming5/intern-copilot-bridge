# J_0024 Ambiguous user request clarification journey

<!-- METADATA:TYPE=J,ID=J_0024,STATUS=Retired,OWNER=intern_ci_lead,SESSION=12 -->

## Retired reason

Session 12 supervisor review rejected this as a J journey. The draft mostly
tests whether the agent decides to ask a clarifying card for an ambiguous
instruction. That is model/prompt behavior unless the product prompt explicitly
requires that action, and generic card behavior is already covered elsewhere.
Do not implement this case unless a future prompt contract explicitly requires
clarification behavior for a concrete product workflow.

## Concise proposal

Prompt: give an intentionally ambiguous work request with two possible target
files or tasks.

Look for: the intern asks a clarifying question instead of guessing.

Do: answer the clarification card/free reply.

Look for: the intern performs only the chosen path and records the decision.

## Goal

Validate the "ask before acting" journey:

- a user request is ambiguous;
- the intern uses `request_user_input`/AskUser rather than guessing;
- the Feishu card/free reply answer is routed to the pending question;
- the intern continues with the chosen option only;
- the final report links the clarification decision to the resulting action.

## Why J

F can validate card shape and callback routing. This case proves a real agent
uses clarification to avoid destructive or wrong work.

## Cost profile

Paid/agent. The case sends a real prompt, answers a real card, and waits for a
real follow-up turn.

## Key assertions

- Ambiguity triggers clarification before file changes.
- The answered choice controls the later action.
- Unchosen target remains unchanged.
- Pending question is closed after the answer.

## Implementation prerequisites

- Real question-card/free-reply driver.
- File diff guard before and after clarification.
- Pending question lifecycle evidence.

## Failure classification

- Intern guesses and edits before asking: product/model behavior finding.
- Clarification answer is ignored or routed wrong: product bug.
- Both files are edited: product bug.
- CI cannot drive the card/free reply: CI helper gap.
