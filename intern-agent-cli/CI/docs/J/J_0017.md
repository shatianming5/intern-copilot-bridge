# J_0017 Plan approval refine-card journey

<!-- METADATA:TYPE=J,ID=J_0017,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=7 -->

## Concise proposal

Prompt: ask Claude to propose a tiny edit plan and wait for approval before
changing files.

Look for: Feishu receives the plan approval card with the expected choices, and
no file edit happens before the user chooses.

Do: choose free reply and ask Claude to narrow the plan.

Look for: Claude refines the plan instead of executing the old one.

Do: choose `reviewEdits` or `acceptEdits`.

Look for: only then does Claude make the tiny edit, and the final result matches
the refined plan.

## Goal

Validate the real plan-approval journey for Claude:

- a user asks Claude to plan before editing;
- the `ExitPlanMode`/plan approval card appears in Feishu;
- free-text feedback refines the plan instead of approving execution;
- an explicit approval choice starts execution;
- file changes match the refined plan and do not happen early.

## Why J

Unit/F coverage can validate card formatting and answer parsing, but it cannot
prove the real supervisor journey where a user reads a plan, sends free-text
feedback, then explicitly approves execution. This case requires a real Claude
turn and a real Feishu card answer.

## Cost profile

Paid/agent. The case starts a Claude session, asks for a plan, waits for a card,
submits a free-text refinement, and then allows a tiny edit.

## Required environment

- Backend: `claude`.
- A case-scoped local/throwaway workspace with one harmless editable file.
- Real Feishu card answer support for plan approval and free-text feedback.
- Redacted Claude runtime evidence; secrets must never appear in reports.

## Key assertions

- No edit happens before explicit approval.
- Free-text feedback refines the plan and does not execute it.
- Approval applies to the refined/latest plan.
- File changes match the refined plan exactly.
- Card lifecycle and answers are visible in Feishu evidence.

## Implementation prerequisites

- Real `ExitPlanMode`/plan card waiter.
- Card free-reply submit helper.
- File diff guard for "no edits before approval" and "only target file changed".
- Redacted Claude prompt/reply evidence.

## Failure classification

- Claude edits before approval: product bug.
- Free reply is treated as approval or ignored: product bug.
- Stale card approval executes the old plan: product bug.
- Card answer cannot be driven through real Feishu callback: CI helper gap.
- Runtime evidence leaks Claude secrets: CI bug.
