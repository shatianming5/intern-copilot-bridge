# J_0061 User interrupts with status request journey

<!-- METADATA:TYPE=J,ID=J_0061,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=9 -->

## Scope note

Keep this case only as running-supplement/product coverage for an active turn
receiving a mid-turn status request. The assertion is delivery, continuity, and
no unintended cancellation, not generic "good progress update" quality.

## Concise proposal

Prompt: start a long harmless task.

Prompt: while it is running, ask "现在进展到哪了？"

Look for: intern gives a concise status update and then continues the original
task unless told to stop.

## Goal

Validate mid-turn status requests:

- long turn is active;
- user asks for status without cancelling;
- intern reports current progress;
- original work continues or resumes afterward;
- final answer includes both status handling and task result.

## Why J

This is a live supervisory conversation over an active agent turn.
