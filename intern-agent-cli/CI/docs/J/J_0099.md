# J_0099 Stop during supplement finalization journey

<!-- METADATA:TYPE=J,ID=J_0099,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=9 -->

## Concise proposal

Prompt: send a supplement near the end of a running turn, then `/stop`.

Look for: no stale finalized supplement overwrites the stopped state, and a
later follow-up is clean.

## Goal

Validate stop and running-supplement ordering:

- a turn is active;
- user sends a supplement while the final answer is near completion;
- user sends `/stop`;
- stale final/supplement output does not appear after stop as if accepted;
- next prompt starts cleanly.

## Why J

Ordering of stop/supplement events can be unit-tested in F, but the user-facing
risk is a confusing Feishu timeline after a real conversation.

## Scenario

| # | Type | Operation | Expected |
|---|------|-----------|----------|
| 1 | action | Start a controlled slow read-only turn. | Turn is active. |
| 2 | action | Send supplement with unique token near finalization. | Supplement is recorded for the active turn. |
| 3 | action | Send `/stop`. | Stop targets the active turn/pending output. |
| 4 | wait/assert | Observe timeline after stop. | No stale final answer claims completed supplement work after stop. |
| 5 | action/wait | Send follow-up with new token. | Clean follow-up answer appears. |

## Key assertions

- Stop wins over stale finalization.
- Follow-up is not polluted by stopped supplement state.

## Implementation prerequisites

- Near-finalization slow-turn fixture.
- Timeline ordering checker.

## Failure classification

- Stale final answer after stop: product bug.
- Follow-up consumes stopped supplement as current instruction: product bug.
