# J_0037 Replaced question stale-card isolation journey

<!-- METADATA:TYPE=J,ID=J_0037,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=8 -->

## Concise proposal

Prompt: make the intern ask one question, then refine so it asks a replacement
question.

Do: click the old card after the replacement exists.

Look for: old card is ignored/stale; only the newest question answer is used.

## Goal

Validate replacement question isolation:

- first question card is created;
- user/refinement causes a second question that supersedes the first;
- old card callbacks cannot answer the new question;
- latest card answer controls the final turn.

## Why J

This is a real user risk: users may click an old card after an updated decision
appears. It requires live card lifecycle evidence.

## Key assertions

- Superseded card cannot answer a newer question.
- Latest pending question remains authoritative.

## Implementation prerequisites

- Replacement-question inducer.
- Card callback lifecycle evidence.

## Failure classification

- Old card answers new question: product bug.
- Replacement leaves two active valid cards: product bug.
