# J_0008 Stop continue screenshot journey

<!-- METADATA:TYPE=J,ID=J_0008,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=4 -->

## Goal

Validate the real user journey for `/stop`, continuation, and `/screenshot` in
an intern Feishu group:

- the user asks an intern to run a controlled long-running probe;
- the user sends `/stop` while the probe is still running;
- the user then says only `你继续`;
- the next turn must not receive the full old prompt or full additional context;
- the intern can continue from conversation/session context;
- `/screenshot` sends a real image message to the group.

## Why J

This is J-scoped because the behavior depends on a real user prompt, a real
agent turn, slash commands delivered in the group, and the agent's continuation
reply. F cases can validate slash routing and low-level stop/screenshot
contracts, but they cannot prove the user-visible "stop, continue, screenshot"
journey after agent work has started.

## Cost profile

Paid/agent. The case starts a real Codex intern session, runs a real turn,
stops it, starts a second real turn with a short continuation prompt, and checks
the screenshot result from Feishu-visible evidence.

## Required environment

- A lead-prepared local or throwaway workspace suitable for long-running probe
  tests.
- Backend: `codex`.
- The workspace mode must be fixed at add time; this case must not add, delete,
  migrate, or switch workspace mode.
- A probe script or fixture equivalent to `scripts/j0008_slow_probe.sh` that:
  - prints `J0008_PROBE_STARTED` soon after launch;
  - runs long enough for `/stop` to arrive before normal completion;
  - emits enough staged output to prove the intern actually started it.

## Implementation prerequisites

Do not weaken this journey into slash API checks. If helpers are missing,
implementation must wait for or add CI helpers for:

- preparing or verifying the long-running probe fixture;
- observing turn start and capturing turn input;
- sending `/stop` as a user-visible slash command;
- waiting for stop application on the active turn;
- verifying continuation input does not contain old full prompt/context;
- waiting for a Feishu image message after `/screenshot`.

## Key assertions

- The slow probe starts before `/stop`, and `/stop` is applied before normal
  probe completion.
- The stopped session can accept a new prompt.
- The continuation turn input contains only the new user prompt `你继续` plus
  normal runtime framing. It must not include the full old long prompt,
  `additionalContext`, or `full additional context` markers.
- The intern's continuation reply is related to the stopped work rather than an
  unrelated new task.
- `/screenshot` produces a real image message with image evidence.
- The case never adds, deletes, migrates, or switches the workspace.

## Failure classification

- `/stop` has no feedback, does not stop the active turn, or leaves the session
  unable to accept prompts: product bug.
- The continuation turn receives the old full prompt or full additional context:
  product bug.
- The intern cannot continue from conversation/session context after `你继续`:
  product bug or model behavior finding, classified by evidence.
- `/screenshot` sends only text, sends to the wrong group, lacks an image key,
  or fails by permission/environment issue: product bug or environment issue,
  classified by report evidence.
- Turn input capture, stop waiter, or screenshot image detection cannot express
  the user-visible path: CI helper gap.
- Workspace cleanup, deletion, migration, or mode switching occurs: CI bug.
