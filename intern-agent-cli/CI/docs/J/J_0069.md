# J_0069 Worker reassignment after failure journey

<!-- METADATA:TYPE=J,ID=J_0069,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=9 -->

## Concise proposal

Prompt: after one worker fails, ask coordinator to reassign to another worker.

Look for: second worker receives the updated context, first worker state is not
overwritten, and coordinator reports both attempts.

## Goal

Validate reassignment after failure:

- first worker failure is retained;
- user asks coordinator to reassign;
- second worker receives context and attempts the task;
- coordinator reports attempt history.

## Why J

This is a real team-management conversation across multiple agents.

## Scenario

| # | Type | Operation | Expected |
|---|------|-----------|----------|
| 1 | action | Create coordinator and two workers with first failure fixture. | Team live. |

## Key assertions

- Reassignment preserves history.
- Worker B receives context, not a blank task.

## Implementation prerequisites

- Multi-worker routing.
- Attempt history evidence.

## Failure classification

- First failure overwritten/lost: product bug.
- Worker B gets wrong/empty context: product bug.
