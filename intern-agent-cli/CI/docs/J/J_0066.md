# J_0066 Team multi-worker delegation aggregation journey

<!-- METADATA:TYPE=J,ID=J_0066,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=9 -->

## Concise proposal

Prompt: ask a coordinator to split a small investigation across two workers.

Look for: two distinct worker tasks/messages, each worker replies, and the
coordinator aggregates both results without flattening roles.

## Goal

Validate multi-worker team delegation:

- coordinator receives a natural request;
- two workers receive distinct scoped work;
- workers reply with separate evidence;
- coordinator aggregates both answers and cites which worker did what.

## Why J

Team CLI/TreeView structures are F. This is the actual user journey for asking
a coordinator to organize multiple agents.

## Scenario

| # | Type | Operation | Expected |
|---|------|-----------|----------|
| 1 | action | Create coordinator and two workers in a case team. | Role/team metadata is explicit. |
| 2 | action | Prompt coordinator to split two read-only checks across workers. | Coordinator plans/delegates. |
| 3 | wait/assert | Wait for worker assignments/messages. | Worker A and B receive distinct tokens/tasks. |
| 4 | action/wait | Prompt or allow workers to complete. | Each worker replies with its token/evidence. |
| 5 | wait/assert | Wait for coordinator aggregation. | Coordinator summarizes both results and attributes each worker. |
| 6 | assert | Check report. | Report links coordinator prompt, worker messages/replies, and aggregation. |

## Key assertions

- Roles remain distinct.
- Both worker results are included and attributed.

## Implementation prerequisites

- Multi-worker team fixture.
- Peer/task delivery evidence.

## Failure classification

- Coordinator does all work itself without delegation: product/model behavior finding.
- Worker result is lost or misattributed: product bug.
