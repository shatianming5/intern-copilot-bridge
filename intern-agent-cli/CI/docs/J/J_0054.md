# J_0054 Stop-hook checklist correction journey

<!-- METADATA:TYPE=J,ID=J_0054,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=9 -->

## Scope note

Keep this case only as product hook coverage: the observable contract is that a
live stop-hook correction produces the required final checklist without mutating
repo/task state. Do not use it to test generic answer polish.

## Concise proposal

Prompt: ask the intern for a response that must include a session checklist.

Do: induce or observe a first response missing required checklist.

Look for: stop hook blocks/corrects the response and the final visible answer
contains the required checklist without changing work state.

## Goal

Validate user-visible stop-hook correction:

- required checklist format is active;
- assistant draft misses required fields;
- stop hook produces corrective feedback;
- final visible response includes the complete checklist;
- no extra code change or merge happens during correction.

## Why J

Hook unit tests can validate templates. This journey proves the live intern
recovers and sends the user a compliant final answer.
