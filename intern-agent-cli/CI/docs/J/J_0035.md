## Retired reason

Session 12 consolidation folded this rich-post mixed-content path into
`J_0021`, the inbound attachment bundle. Rich text plus image/file ordering and
honest unsupported image handling are now covered as Phase 4 of `J_0021`.

## Concise proposal

Do: send a Feishu rich post containing text plus an inline image attachment.

Prompt: ask the intern to combine the post text and image evidence.

Look for: text and inline image are both routed to the intern, and the reply
does not treat the image as missing when it was downloaded.

## Goal

Validate real Feishu `post` mixed-content handling:

- post text is extracted as prompt-visible text;
- inline image elements are extracted as attachments;
- the intern sees both the text and attachment path;
- the reply combines both sources or honestly reports unsupported image reading.

## Why J

`extract_text_from_message` and `extract_attachments` are F-level. This case
proves a real rich post is usable in a user conversation.
