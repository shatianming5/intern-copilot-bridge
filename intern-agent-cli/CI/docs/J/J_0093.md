# J_0093 Coordinator owner-filter status journey

<!-- METADATA:TYPE=J,ID=J_0093,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=9 -->

## Scope note

Keep this case only as team ownership/product-scope coverage. It should assert
that the coordinator-visible status is filtered by explicit ownership metadata,
not judge generic status-summary quality.

## Concise proposal

Prompt: ask a coordinator to summarize owned teams and workers.

Look for: status includes only resources owned by that coordinator and clearly
omits unrelated teams.

## Goal

Validate coordinator owner-filter visibility:

- coordinator has one owned team and at least one unrelated team exists;
- user asks for a status rollup;
- coordinator summarizes only owned team/workers;
- unrelated team names/tasks are absent or explicitly marked out of scope.

## Why J

Tree ownership filters can be F. The supervisor journey is asking a coordinator
what it owns and getting a safe, scoped answer.
