# J_0100 Policy restart resume shell continuity journey

<!-- METADATA:TYPE=J,ID=J_0100,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=9 -->

## Concise proposal

Prompt: trigger a policy restart during a normal conversation, then continue.

Look for: restart/resume does not leave the session in a shell/background state,
and the same Feishu group can keep working.

## Goal

Validate policy restart user continuity:

- a real agent turn is running or just completed;
- policy/env update requires restart;
- restart/resume happens without exposing a broken shell state to the user;
- follow-up prompt receives an agent answer in the same group.

## Why J

Restart command shape and process state are F. This case proves the supervisor's
conversation remains usable after policy-driven restart.

## Scenario

| # | Type | Operation | Expected |
|---|------|-----------|----------|
| 1 | action | Start a normal Codex conversation with token. | Agent replies or remains active. |
| 2 | action | Trigger a case-scoped policy/env update requiring restart. | Restart requirement is recorded. |

## Key assertions

- Restart does not strand the session in shell/background state.
- Same group remains usable after restart.

## Implementation prerequisites

- Safe policy marker driver.
- Session readiness and follow-up verifier.

## Failure classification

- Restart leaves shell instead of agent: product bug.
- Follow-up lost after restart: product bug.
