# J_0015 Feishu group mode live-turn journey

<!-- METADATA:TYPE=J,ID=J_0015,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=7 -->

## Concise proposal

Do: put a Codex intern group in trigger-all mode.

Prompt: send a normal message and ask for a tiny tool-backed reply.

Look for: the intern wakes and replies.

Do: switch the group to at-only mode.

Prompt: send a normal message, then an @/mention message.

Look for: the normal message does not start a turn; the @/mention message does.

Do: switch detail mode between summary and full during tool-backed turns.

Look for: summary suppresses noisy tool blocks while keeping visible progress;
full shows the richer tool detail. If no-collapse is enabled for a long reply,
the group receives continuation messages rather than one silently collapsed
message.

## Goal

Validate user-visible group mode behavior during real Codex turns:

- `trigger_mode=all` wakes on normal group messages;
- `trigger_mode=at_only` does not wake on non-mention group chatter;
- mention/explicit target still wakes in at-only mode;
- `detail_mode=summary` changes tool rendering without hiding final progress;
- `detail_mode=full` restores detailed tool rendering;
- `no_collapse_mode=on` preserves long output by sending continuation messages.

## Why J

F cases validate `/config`, slash/card handlers, GUI context commands, daemon
proxies, and persistent group config. This case is J-scoped because it checks
whether those settings affect a real user conversation with a live Codex intern.

## Cost profile

## Key assertions

- All/at-only behavior is proven by real turn start or absence of turn start.
- Non-mention chatter in at-only mode does not trigger agent work.
- Addressed message in at-only mode still triggers agent work.
- Summary mode hides noisy tool detail without hiding the user's final answer.
- Full mode restores detailed tool rendering.
- No-collapse mode produces ordered continuation messages for long output.

## Implementation prerequisites

- Real Feishu mention/addressed-message driver or an accepted equivalent.
- No-turn waiter that can prove absence of a new turn over a bounded window.
- Timeline classifier for summary/full/no-collapse rendering differences.

## Failure classification

- At-only mode still wakes on normal group chatter: product bug.
- Addressed messages do not wake the intern in at-only mode: product bug or
  missing Feishu mention driver, classified by evidence.
- Summary mode hides final progress or full mode fails to restore detail:
  product bug.
- No-collapse mode loses or reorders continuation messages: product bug.
- Absence-of-turn evidence is too weak: CI helper gap.
