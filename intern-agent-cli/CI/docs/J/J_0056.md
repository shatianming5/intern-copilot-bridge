# J_0056 User asks for evidence path journey

<!-- METADATA:TYPE=J,ID=J_0056,STATUS=Retired,OWNER=intern_ci_lead,SESSION=12 -->

## Retired reason

Session 12 scope audit retired this case. Asking for evidence paths is a
generic reporting instruction unless tied to a specific product/built-in prompt
contract; it should not be a standalone J journey.

## Concise proposal

Prompt: after a task turn, ask "给我 evidence 路径和关键输出，不要泛泛总结".

Look for: intern returns concrete report/log/test paths and the important
observed lines, not vague claims.

## Goal

Validate evidence-oriented reporting:

- intern performs a small check or uses existing report artifact;
- user asks for evidence paths;
- reply includes concrete local paths/message ids and short key output;
- no fabricated path is reported.

## Why J

This is a supervisor communication journey. F can validate report files exist,
but not that the agent reports them usefully.
