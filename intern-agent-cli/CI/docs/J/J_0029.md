# J_0029 Compaction preserves task state journey

<!-- METADATA:TYPE=J,ID=J_0029,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=8 -->

## Concise proposal

Prompt: assign a tiny task and have Codex summarize its plan and current file.

Do: force compaction/resume.

Prompt: ask Codex to continue without restating the task details.

Look for: it resumes the same task, current branch, and newest user instruction
instead of acting like a fresh unrelated chat.

## Goal

Validate task continuity across compaction/resume:

- a real task is assigned through Feishu;
- the intern starts work and records task/branch context;
- compaction/resume occurs;
- a short "continue" prompt resumes the same task;
- final evidence points to the same task and does not regress to stale context.

## Why J

F can validate metadata and compaction markers. This case proves a user can keep
working with the same intern after context compaction without re-explaining the
task.

## Cost profile

Paid/agent. The case uses a real Codex task turn and compaction/resume boundary.

## Key assertions

- Compaction/resume preserves task identity and branch context.
- Short continue prompt is enough after compaction.
- Newest user instruction still wins.

## Implementation prerequisites

- Compaction/resume helper.
- Task assignment and status readers.
- Transcript evidence across compaction boundary.

## Failure classification

- Intern forgets assigned task after compaction: product/model behavior finding.
- Resume starts a fresh unrelated session: product bug.
- Old instruction overrides new continue prompt: product bug.
- CI cannot force/observe compaction safely: CI helper gap.
