# J_0068 Worker failure escalation journey

<!-- METADATA:TYPE=J,ID=J_0068,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=9 -->

## Concise proposal

Prompt: make one worker hit a controlled failure.

Look for: worker reports the blocker with evidence, coordinator surfaces it to
the user, and no fake success is reported.

## Goal

Validate failure escalation in team mode:

- worker hits a known impossible or failing fixture;
- worker reports concrete blocker evidence;
- coordinator aggregates the failure honestly;
- user sees failure classification and next action.

## Why J

The user journey is about truthful escalation across agents.

## Key assertions

- Failure is not hidden.
- Coordinator does not convert failure into success.

## Implementation prerequisites

- Controlled failure fixture.
- Team aggregation evidence.

## Failure classification

- Fake success despite worker failure: product/model behavior bug.
