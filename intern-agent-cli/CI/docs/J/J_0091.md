# J_0091 Project-scoped evidence path reporting after failure journey

<!-- METADATA:TYPE=J,ID=J_0091,STATUS=Retired,OWNER=intern_ci_lead,SESSION=12 -->

## Retired reason

Session 12 scope audit retired this case. Asking an intern to report exact
evidence paths after a failure is generic closeout/reporting behavior unless it
is tied to a specific product or built-in prompt contract. Project/session
isolation for paths and transcripts remains covered by `J_0092`; report file
layout and path-scope helpers belong in F/source-contract coverage.

## Concise proposal

Prompt: ask for failure evidence after a controlled failing command.

Look for: final answer gives exact project/run-scoped report paths and key
lines, not vague "logs are available" text.

## Goal

Validate evidence path closeout:

- controlled failure creates logs/reports under the case scope;
- user asks for evidence path;
- agent replies with exact local paths and key result lines;
- paths include project/run dimensions and do not point to stale sessions.

## Why J

Log file layout is F. The user journey is the supervisor asking "where is the
evidence?" and receiving useful, scoped paths.
