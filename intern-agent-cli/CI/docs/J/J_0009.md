# J_0009 Codex goal lifecycle and Feishu confirmation journey

<!-- METADATA:TYPE=J,ID=J_0009,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=6 -->

## Concise proposal

Prompt: send `/goal finish a small visible checklist`.

Look for: Feishu reports the goal as active, the Codex pane shows the same
objective, and later user turns show a compact goal footer without duplicate
active events.

Prompt: send `/goal finish a different checklist`.

Look for: Codex shows `Replace goal?`, Feishu sends a confirmation card, and
choosing `Cancel` keeps the old objective.

Prompt: send the second goal again and choose `Replace current goal`.

Look for: the new objective becomes active, exactly one replacement lifecycle
message is visible, `/goal status` reports the new objective, and `/goal clear`
clears it.

## Goal

Validate the real user journey for Codex goal control from Feishu:

- set a goal from a Feishu slash command;
- observe active goal state in Feishu and the Codex pane;
- trigger the Codex replace confirmation UI;
- drive the replacement through a Feishu card;
- verify goal status, continuation rendering, and clear behavior.

## Why J

The codebase has API and hook tests for goal delivery, Codex panel fallback,
replace confirmation detection, Feishu cards, synthetic continuation, and
footer/lifecycle de-duplication. This case is J-scoped because it stitches
those pieces into the actual supervisor journey: the user sends `/goal`, sees a
confirmation card, chooses Cancel or Replace, and then sees the goal state
reflected across Feishu messages and Codex runtime state.

## Cost profile

Paid/agent. The case starts a real Codex intern session, sends goal commands,
lets Codex enter active goal state, and may trigger goal continuation turns.

## Required environment

- Backend: `codex`.
- A case-scoped Feishu group and Codex session.
- Real card answer support for the goal replacement confirmation.
- Turn/goal evidence from Codex transcript, goal sqlite snapshot, tmux pane, or
  equivalent product source.

## Key assertions

- Active goal evidence matches the Feishu objective and is not inferred from a
  pending input line alone.
- Replace confirmation is surfaced as a Feishu card and waits for user choice.
- Cancel preserves the old goal; Replace activates the new goal.
- Goal lifecycle messages are not duplicated across post-tool/stop/continuation
  rendering.
- `/goal clear` clears the active goal and future turns do not revive stale
  state.

## Failure classification

- `/goal` delivery reports failure while pane/source evidence shows success:
  product bug.
- Replace UI appears but no Feishu card is sent: product bug.
- Card answer is not delivered to the Codex pane: product bug or CI helper gap.
- Goal lifecycle events duplicate or appear out of order: product bug.
- Goal source evidence cannot be captured: CI helper gap.
