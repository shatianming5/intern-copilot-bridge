## Retired reason

Supervisor review in Session 10 rejected this as a generic LLM behavior test,
not a product/plugin or built-in-prompt journey. Broad "review this diff",
"fix a bug", or "answer a general question" scenarios are outside this J set
unless the expected behavior is specifically created by the intern plugin,
Feishu integration, slash command, session/goal/helper/peer flow, or another
owned prompt contract.

## Concise proposal

Prompt: ask the intern to review a small diff and not edit files.

Look for: findings lead, file/line references are concrete, and no file changes
or branches/MRs are created.

## Goal

Validate review-only behavior:

- user asks for a code review, not implementation;
- intern inspects the diff;
- response starts with findings and evidence;
- no code/metadata files are modified;
- report records the no-edit boundary.

## Why J

This is a user-facing agent mode: review stance differs from implementation
stance and depends on natural-language instruction.
