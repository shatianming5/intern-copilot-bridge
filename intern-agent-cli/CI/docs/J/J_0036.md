# J_0036 Question card timeout and re-ask journey

<!-- METADATA:TYPE=J,ID=J_0036,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=8 -->

## Concise proposal

Prompt: make the intern ask a decision question.

Do: let the card expire or mark it stale via the accepted test control.

Prompt: ask the intern to ask again.

Look for: the stale card cannot answer the new question; the fresh card can.

## Goal

Validate stale/timeout question recovery:

- one pending question appears;
- timeout/stale state is visible;
- old answers are rejected after timeout;
- the intern can re-ask and receive a fresh answer;
- final work uses only the fresh answer.

## Why J

Card timeout source rules are F; the user journey is recovering from a missed
decision and answering the re-asked question.

## Key assertions

- Stale card cannot mutate current pending state.
- Re-ask path works.
- Fresh answer controls final behavior.

## Implementation prerequisites

- Timeout/stale-card test control or bounded timeout.
- Card answer driver and pending state reader.

## Failure classification

- Stale answer is accepted: product bug.
- Re-ask cannot create a fresh card: product bug.
- CI cannot safely accelerate timeout: CI helper gap.
