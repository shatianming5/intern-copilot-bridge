# J_0096 Machine helper saved profile reconnect journey

<!-- METADATA:TYPE=J,ID=J_0096,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=9 -->

## Concise proposal

Prompt: start a machine helper, stop/reconnect, then ask a follow-up.

Look for: the helper resumes against the intended saved machine profile or
clearly reports that reconnect is unavailable.

## Goal

Validate helper machine continuity:

- helper starts against a selected/saved machine profile;
- user stops or loses the helper session;
- follow-up reconnect targets the same intended machine profile;
- user sees success or a precise reconnect blocker.

## Why J

Profile storage and helper action routes are F. This case verifies the helper
conversation continuity a user experiences.

## Scenario

| # | Type | Operation | Expected |
|---|------|-----------|----------|
| 1 | action | Start helper on a case machine profile. | Helper group is usable. |
| 2 | action | Prompt helper for machine marker/status. | Reply proves intended machine/profile. |
| 3 | action | Stop helper or simulate reconnect window. | Binding is no longer active. |
| 4 | action/wait | Restart/reconnect helper and ask follow-up. | Same profile is used, or blocker is precise. |
| 5 | assert | Check report. | Report links profile id, helper replies, and reconnect result. |

## Key assertions

- Reconnect does not silently switch machines.
- Unsupported reconnect is reported honestly.

## Implementation prerequisites

- Case machine profile fixture.
- Helper group timeline and profile evidence.

## Failure classification

- Wrong machine after reconnect: product bug.
- Fake reconnect success: product/model behavior bug.
