# J_0043 Claude real-turn exit resume same-session journey

<!-- METADATA:TYPE=J,ID=J_0043,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=4 -->

## Goal

Validate the user-visible Claude resume journey after a real Claude turn:

- create and start a `type=claude` intern session in a case-scoped Feishu group;
- send a minimal natural prompt and wait for a Claude reply;
- capture durable Claude session UUID evidence only after that real turn;
- send `/exit`, capture the printed `Resume this intern` command, execute it,
  and require the same UUID;
- run the GUI-equivalent restart command and require `restarted via resume
  <uuid>` with the same UUID.

## Why J

`F_0043` validates Claude intern lifecycle, group type, no-prompt start, `/exit`
hint shape, and restart/resume contracts. It intentionally does not send a
natural user prompt. This case is J-scoped because it proves the stricter user
journey: after Claude has actually replied to a user, `/exit`, manual resume,
and GUI-equivalent restart must keep the same durable Claude session.

## Cost profile

Paid/agent. The case starts a real Claude session, sends a real prompt, waits
for Claude output, and records only redacted provider/token evidence.

## Required environment

- Backend: `claude`.
- A lead-prepared local or throwaway workspace suitable for Claude lifecycle
  tests.
- Claude provider token/policy material may be loaded only through the approved
  policy alias used by F Claude cases. Secret values must never be written to
  docs, stdout, reports, or retained artifacts.
- The workspace mode must be fixed at add time; this case must not add, delete,
  migrate, or switch workspace mode.

## Case resources

- namespace: `ci_j_0043`
- intern: `intern_ci_j_0043_claude_<run_id>`
- task fixture: `task_ci_j_0043_resume_<run_id>`
- report prefix: `j_0043_claude_resume`
- retained scene: the final Feishu group, prompt/reply messages, redacted
  Claude runtime evidence, resume hint, restart output, UUID comparison
  evidence, and CI report artifacts remain available for review.

Start-of-case cleanup may remove only the `ci_j_0043` namespace from previous
runs. It must not clean lead-owned workspaces, shared debug deployments, or
other cases' resources.

## Implementation prerequisites

Do not weaken this journey into no-prompt lifecycle checks. If helpers are
missing, implementation must wait for or add CI helpers for:

- starting and observing a real Claude turn;
- waiting for a Claude reply to a natural user prompt;
- capturing durable Claude session UUID evidence after the real turn;
- redacting Claude token, policy, and runtime evidence in reports;
- validating `/exit` resume hint and restart UUID continuity for `type=claude`.

## Key assertions

- Claude type does not drift in status/list, session map, group registry, or
  report evidence.
- Same-session UUID is required only after the real Claude reply has completed.
- The `/exit` hint command includes both `--project` and `--type claude`.
- Manual resume and GUI-equivalent restart reuse the durable Claude UUID.
- Restart output must say `restarted via resume <uuid>`; fresh restart is a
  failure after the real turn.
- Token, policy, and runtime evidence are redacted.
- The case never adds, deletes, migrates, or switches the workspace.

## Failure classification

- Claude cannot start or type drifts to another backend: product bug.
- Claude reply cannot be observed for a natural prompt: CI helper gap or product
  bug, classified by evidence.
- Durable Claude UUID cannot be discovered after a real turn: capability gap.
- `/exit` hint is missing, malformed, lacks `--project` or `--type claude`, or
  cannot be executed: product bug.
- Manual resume or restart starts fresh or changes UUID after the real turn:
  product bug.
- Secret provider token or policy material appears in stdout/report/artifacts:
  CI bug.
- Workspace cleanup, deletion, migration, or mode switching occurs: CI bug.
