# F_0012 Daemon group proxy and registry mutation

<!-- METADATA:TYPE=F,ID=F_0012,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=2 -->

## 目标

验证 daemon group proxy API：group create/delete 请求必须带 project scope，正确调用 relay，并同步本机 registry；`/api/group/sync` 在 enterprise mode 下必须返回显式 disabled contract。该用例不要求 agent 响应群消息。

## 资源

- workspace/project: `ci_f_0012_project_{run_id}`。
- intern: `intern_ci_f_0012_worker_{run_id}`。

## 通过标准

- create/delete 都保留 project scope。
- `/api/group/sync` 返回 HTTP 410 disabled contract，且不破坏已有 registry。
- daemon registry 与 relay registry 在 create/delete 后最终一致。
- 缺 project 请求失败，不产生 default/global 群。

## 失败分类

- project 丢失或串到同名其他 workspace：产品 bug。
- create 失败但 registry 写入成功：产品 bug。
- delete 删除其他 project 同名 intern：产品 bug。
- `/api/group/sync` 返回 200 或修改 registry：产品 contract 变化，需要先和主管确认再改剧本。
