# F_0022 Workspace enable re-enable doctor and TreeView refresh

<!-- METADATA:TYPE=F,ID=F_0022,STATUS=ApprovedForImplementation,OWNER=intern_ci_lead,SESSION=13 -->

## 目标

验证 workspace enable/disable/re-enable、`workspace doctor`、`intern.refreshTree` 与 TreeView 投影一致：disabled workspace 隐藏其 interns/tasks/skills，再次 enable 后恢复显示，refreshTree 能同步 metadata 并报告错误。

## 资源

- debug machine: `debug-a` 或 `debug-b`。
- workspace display: `ci_f_0022_workspace_{run_id}`。
- intern: `intern_ci_f_0022_{run_id}`。
- task: `task_ci_f_0022_open_{run_id}`。
- skill package: `ci_f_0022_skill_source_{run_id}`。

## 通过标准

- disable 后 workspace 及其 intern/task/skill 从 TreeView 消失，但 definition 不被删除。
- re-enable 后 TreeView 投影恢复，metadata 不丢失。
- `workspace doctor` 能给出健康报告。
- refreshTree 的 metadata sync 失败可见、可分类，并保留 last-good TreeView 状态。

## 失败分类

- disabled workspace 仍可在 TreeView 操作：产品 bug。
- re-enable 后 metadata 丢失：产品 bug。
- refreshTree 静默吞掉 sync 失败：产品 bug。
- 注入失败污染非本 case workspace：CI 逻辑错误。
