# F_0055 Team create rollback and retry contract

<!-- METADATA:TYPE=F,ID=F_0055,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=7 -->

## 目标

验证 `internctl team create` / VS Code Create Team 在 Phase 2 失败时不会留下
半创建 team：已经写入的 team/member metadata、Feishu groups、session registry、
runtime dirs 和 relay registry 必须被 rollback；错误输出必须保留真实失败步骤；
清理后同一 namespace 可以成功 retry。

本 case 不要求 coordinator、lead 或 worker 执行自然语言任务；真实 team 委派旅
程由 `J_0010` 覆盖。

早期 TreeView F 包曾明确不做 Team 功能；本文件只是后续 reliability
ReviewDraft。实现前必须由主管重新批准 Team create rollback 覆盖范围。

## 通过标准

- Phase 2 失败时返回非 0，并展示可诊断的真实失败步骤。
- 已创建的 metadata、Feishu groups、relay registry、session registry 和 runtime
  dirs 都被自动 rollback。
- TreeView/CLI list 不留下 ghost team 或 ghost members。
- 移除 failure hook 后同一 namespace 能成功 retry。
- Retry 后 `team delete --force` 能清理本 case team，不误删其它 workspace/team。
