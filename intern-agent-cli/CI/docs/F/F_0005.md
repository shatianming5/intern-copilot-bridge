# F_0005 Workspace duplicate and invalid add rollback

<!-- METADATA:TYPE=F,ID=F_0005,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=2 -->

## 目标

验证 workspace add 的错误路径：同名重复、坏 repo URL、坏 branch、缺 provider 参数都必须失败并自动回滚，不得留下半成品 registry、metadata branch 或 relay record。

## 资源

- base workspace: `ci_f_0005_base_{run_id}`。
- invalid display names: `ci_f_0005_bad_*`。

## 剧本

| # | 类型 | 操作 | 注释 |
|---|------|------|------|
| 1 | action | `action_reset_case_namespace(case_id="F_0005")` | 初始化错误回滚 case 的 namespace，确保后续失败断言只观察本 case 资源。 |
| 2 | action | `action_cli_workspace_create(provider="codeup", repo_url=valid_repo, mode="repo_dotdir", display_name=base_display)` | 先创建一个合法 baseline workspace，用来触发重复名称错误。 |

## 通过标准

- 所有错误路径都有结构化错误。
- 失败请求不留下 registry、state dir、metadata branch 或 relay record。
- base workspace 仍可正常删除。

## 失败分类

- 半成品状态残留：产品 bug。
- 错误只表现为 traceback 或 timeout：产品 bug。
- invalid case 触碰真实非 case workspace：CI 逻辑错误。
