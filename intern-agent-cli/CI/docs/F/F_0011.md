# F_0011 Daemon status and readiness API

<!-- METADATA:TYPE=F,ID=F_0011,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=2 -->

## 目标

验证部署后 daemon 的基本 status/readiness API：CLI status、HTTP status、relay connection、runtime policy 和 machine identity 必须一致。

## 资源

- machines: `debug-a`、`debug-b`。
- work root: `/root/axis_enterprise_ci`。

## 剧本

| # | 类型 | 操作 | 注释 |
|---|------|------|------|
| 1 | action | `action_require_deploy_gate_passed(report_path=deploy_report)` | 确认部署阶段已通过，F case 只能在 relay/daemon/LB 配置都完成后执行。 |
| 2 | action | `action_remote_cli(machine, "internctl daemon status --json")` | 在目标 debug 机器上执行 daemon status CLI，获取用户可见的 daemon 状态。 |
| 3 | assert | `assert_daemon_status_payload(running=True, relay_connected=True, machine_id=machine_id)` | 断言 CLI JSON 显示 daemon running、已连接 relay，并且 machine_id 是当前机器。 |
| 4 | action | `action_remote_http(machine, "GET /api/status")` | 直接访问 daemon HTTP status API，验证 CLI 之外的 API 入口。 |
| 5 | assert | `assert_daemon_http_status(running=True, machine_id=machine_id)` | 断言 HTTP status 与 CLI status 的 running 和 machine identity 一致。 |
| 6 | assert | `assert_daemon_policy_materialized(provider="codex", lb_mode="enabled")` | 断言 daemon 已按部署要求落地 Codex LB provider 模式，而不是使用登录账号默认 quota。 |
| 7 | assert | `assert_daemon_user_env_secret_present(secret_name="CODEX_LB_API_KEY", redacted_in_report=True)` | 断言 LB secret 在远端用户环境可用，同时 report 中必须脱敏。 |
| 8 | repeat | `debug-a` and `debug-b` | 对两台固定 debug 机器都执行步骤 2 到步骤 7，确保部署不是单机偶然成功。 |
| 9 | action | `action_remote_cli(debug_a, "intern-adminctl relay status --json")` | 在 relay 所在机器读取 relay status，验证 relay 侧机器连接视图。 |
| 10 | assert | `assert_relay_machine_connections(expected=2)` | 断言 relay 看到两台 daemon 连接，作为 F 阶段后续跨机器 case 的基础。 |

## 通过标准

- 两台 daemon CLI/HTTP status 均 running。
- 两台 daemon 均 relay_connected。
- daemon policy/runtime/env 与 deploy gate 一致。
- relay 看到两台机器连接。

## 失败分类

- status running 但 relay_connected false：产品 bug 或 deployment failure。
- policy/env 缺失：deployment bug。
- secret 明文进入 report：产品 bug。
