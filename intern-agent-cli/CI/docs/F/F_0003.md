# F_0003 GitHub workspace add/remove

<!-- METADATA:TYPE=F,ID=F_0003,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=2 -->

## 目标

验证 GitHub workspace 在受控 debug 环境中的添加、provider 识别、metadata mode 和删除。该用例只测插件 workspace 功能，不创建 PR，不启动 agent。

## 资源

- repo: `INTERN_CI_GITHUB_NONPROTECTED_REPO` / `ENTERPRISE_CI_GITHUB_TEST_REPO`，未配置时使用 CI 默认 GitHub 非保护仓库 `git@github.com:chlxydl/intern_debug_repo.git`。
- workspace display: `ci_f_0003_github_{run_id}`。
- modes: `repo_dotdir` 和 `metadata_branch` 各一轮，或按主管 review 后拆成两个 case。

## 剧本

| # | 类型 | 操作 | 注释 |
|---|------|------|------|
| 1 | action | `action_reset_case_namespace(case_id="F_0003")` | 初始化 GitHub workspace case 的 namespace，只清理本 case 的 workspace、metadata 和 relay 记录。 |
| 2 | action | `action_resolve_github_test_repo()` | 解析 GitHub 测试仓库：优先读 `INTERN_CI_GITHUB_NONPROTECTED_REPO` / `ENTERPRISE_CI_GITHUB_TEST_REPO`，否则使用硬编码 CI 默认仓库。 |
| 3 | action | `action_gui_add_workspace(provider="github", repo_url=github_repo, mode="repo_dotdir", display_name=display_dotdir)` | 模拟 GUI 添加 GitHub `repo_dotdir` workspace，覆盖 GitHub provider 的第一种存储模式。 |
| 4 | assert | `assert_workspace_record(provider="github", mode="repo_dotdir", repo_url=github_repo)` | 断言 registry 中 GitHub provider、repo_dotdir mode 和 repo_url 写入正确。 |
| 5 | assert | `assert_relay_workspace_sync(display_name=display_dotdir, provider="github")` | 断言 GitHub workspace 也会同步到 relay，不只 Codeup provider 生效。 |
| 6 | action | `action_gui_add_workspace(provider="github", repo_url=github_repo, mode="metadata_branch", display_name=display_meta)` | 模拟 GUI 添加同一 GitHub 仓库的 metadata_branch workspace，覆盖第二种存储模式。 |
| 7 | assert | `assert_workspace_record(provider="github", mode="metadata_branch", repo_url=github_repo)` | 断言 metadata_branch workspace 在 registry 中独立存在且 mode 正确。 |
| 8 | assert | `assert_metadata_branch_created(workspace=display_meta, provider="github")` | 断言 GitHub metadata branch 创建成功，验证 GitHub provider 的 metadata 写入路径。 |
| 9 | action | `action_cli_workspace_delete(display_name=display_dotdir, confirm=True)` | 删除 GitHub repo_dotdir workspace，验证 CLI 能清理由 GUI 创建的 GitHub workspace。 |
| 10 | action | `action_cli_workspace_delete(display_name=display_meta, confirm=True)` | 删除 GitHub metadata_branch workspace，避免后续 case 复用同一仓库时名称冲突。 |
| 11 | assert | `assert_workspace_removed(display_name=display_dotdir)` | 断言 repo_dotdir workspace 已从本机 registry 删除。 |
| 12 | assert | `assert_workspace_removed(display_name=display_meta)` | 断言 metadata_branch workspace 已从本机 registry 删除。 |

## 通过标准

- GitHub provider 不被误判为 Codeup/local。
- 两种 mode 的 workspace record、metadata root 和 relay sync 均正确。
- 删除不影响同 repo 的另一 workspace 记录。

## 失败分类

- GitHub 仓库凭据不可用：environment/external。
- provider 识别错误或 metadata branch 写错位置：产品 bug。
- 删除相互串扰：产品 bug。
