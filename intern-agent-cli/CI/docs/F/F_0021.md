# F_0021 Workspace disable vs delete GUI contract

<!-- METADATA:TYPE=F,ID=F_0021,STATUS=ApprovedForImplementation,OWNER=intern_ci_lead,SESSION=13 -->

## 目标

验证 VS Code TreeView workspace 右键的 `Remove Workspace` 与 `Stop Maintaining Project` 是两个不同 GUI 合约：前者只在本机 disable workspace，后者删除 relay/workspace registry 里的 workspace 定义。覆盖确认取消、typed name mismatch、`axis_intern_agents` 正常 remove/stop-maintain 路由，以及 GUI 到 CLI 的精确映射。

## 资源

- debug machine: `debug-a` 或 `debug-b`。
- workspace display:
  - `ci_f_0021_disable_{run_id}`。
  - `ci_f_0021_delete_{run_id}`。
- provider: `local` 或 CI 专用 nonprotected provider。
- policy-sensitive workspace: `axis_intern_agents`；不再作为 protected workspace，source/fixture contract 验证它走与普通 workspace 相同的 disable/delete 路由。

## 通过标准

- `Remove Workspace` 只 disable 本机 workspace，保留 relay/workspace definition。
- `Stop Maintaining Project` 只有在两层确认成功后才 delete workspace definition。
- 取消、typed mismatch 都不调用破坏性 CLI；`axis_intern_agents` 不再有特殊保护，必须走普通 disable/delete 映射。
- GUI/CLI contract 可以精确表达；若 GUI 行为无法对应 CLI，应作为产品 bug 报告。

## 失败分类

- GUI remove 调用 delete 或 stop-maintain 调用 disable：产品 bug。
- 取消/typed mismatch 仍修改 registry：产品 bug。
- 测试触碰非 `ci_f_0021_` namespace 或真实用户 workspace 状态：CI 逻辑错误。
