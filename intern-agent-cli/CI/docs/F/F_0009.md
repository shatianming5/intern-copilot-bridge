# F_0009 Codex session lifecycle without task prompt

<!-- METADATA:TYPE=F,ID=F_0009,STATUS=ApprovedForImplementation,OWNER=intern_ci_lead,SESSION=62 -->

## 目标

验证 Codex session 的 start、restart 和 status lifecycle。该用例只验证 session 管理和 tmux/runtime 状态，不要求 agent 完成任何任务；no-prompt 条件下 restart 走 fresh start 是正常结果。

## 资源

- workspace: `ci_f_0009_workspace_{run_id}`。
- intern: `intern_ci_f_0009_codex_{run_id}`。

## 通过标准

- GUI start/restart 与 CLI session command 等价。
- session running、tmux target、status JSON 可判定。
- no-prompt restart 后 session/tmux/status running；不要求 Codex session id。

## 失败分类

- GUI restart 调不存在 CLI：产品 bug。
- status 显示 running 但 tmux 不存在：产品 bug。
- restart 后 registry 指向旧 dead session：产品 bug。
- restart 后 session/tmux 不 running：产品 bug。
