# F_0018 Helper machine selection, detailed mode, stop

<!-- METADATA:TYPE=F,ID=F_0018,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=2 -->

## 目标

验证 helper 控制卡的机器选择、detailed mode 保存和 stop 行为。该用例依赖 `F_0017` 的 helper card/action helper，可与 `F_0017` 分给同一 worker。

## 资源

- project: `ci_f_0018_project_{run_id}`。
- machines: `debug-a`、`debug-b`。

## 剧本

| # | 类型 | 操作 | 注释 |
|---|------|------|------|
| 1 | action | `action_reset_case_namespace(case_id="F_0018")` | 初始化 helper machine/detail/stop case 的 namespace，只清理本 case helper 配置和 runtime。 |
| 2 | action | `action_seed_group(project=project, intern=intern)` | 构造已绑定 group，作为 helper 卡片操作的上下文。 |
| 3 | action | `action_send_slash(chat_id=chat_id, text="/helper")` | 发送 `/helper` 打开控制卡，模拟用户进入 helper 管理界面。 |
| 4 | wait | `wait_for_helper_card(chat_id=chat_id)` | 等待 helper 卡片出现，确保后续点击都有有效 card token。 |
| 5 | action | `action_click_helper_card(action="select_machine", value="debug-b")` | 模拟用户在卡片里选择 `debug-b` 作为 helper 运行机器。 |
| 6 | assert | `assert_helper_machine_selected(project=project, machine="debug-b")` | 断言机器选择被保存到 project helper 配置。 |
| 7 | action | `action_click_helper_card(action="set_detail_mode", value="detailed")` | 模拟用户把 helper detail mode 改成 detailed。 |
| 8 | assert | `assert_helper_detail_mode_saved(project=project, detail_mode="detailed")` | 断言 detailed mode 持久化，避免卡片显示和后端配置不一致。 |
| 9 | action | `action_click_helper_card(action="start")` | 点击 start，要求 helper 按已选择机器和 detail mode 启动。 |
| 10 | wait | `wait_for_helper_running(project=project, machine="debug-b")` | 等待 helper 在 `debug-b` 上进入 running。 |
| 11 | assert | `assert_helper_running_on_machine(project=project, machine="debug-b", detail_mode="detailed")` | 断言 helper 运行机器和 detail mode 与用户卡片选择一致。 |
| 12 | action | `action_click_helper_card(action="stop")` | 模拟用户点击 stop 停止 helper。 |
| 13 | wait | `wait_for_helper_stopped(project=project)` | 等待 helper runtime 完全停止。 |
| 14 | assert | `assert_helper_stopped(project=project, no_route_after_stop=True)` | 断言 helper 已停止且后续消息不会继续路由到已停止 helper。 |

## 通过标准

- 机器选择持久化并影响 helper start。
- detailed mode 保存并进入 helper runtime config。
- stop 后 helper 不再接收路由。

## 失败分类

- 选择 debug-b 但 helper 跑在 debug-a：产品 bug。
- detailed mode 只在卡片显示、不进入 runtime：产品 bug。
- stop 返回成功但 helper 仍在线：产品 bug。
