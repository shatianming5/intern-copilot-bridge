# F_0002 Codeup metadata_branch workspace add/remove

<!-- METADATA:TYPE=F,ID=F_0002,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=2 -->

## 目标

验证 Codeup `metadata_branch` workspace 的 GUI/CLI 添加和删除合约，重点检查 metadata branch 初始化、branch evidence 和删除不污染业务 branch。

## 资源

- repo: CI 默认 nonprotected Codeup repo。
- workspace display: `ci_f_0002_codeup_meta_{run_id}`。
- mode: `metadata_branch`。

## 剧本

| # | 类型 | 操作 | 注释 |
|---|------|------|------|
| 1 | action | `action_reset_case_namespace(case_id="F_0002")` | 初始化本 case namespace，只删除 `F_0002` 前缀资源和历史 metadata，保留其他现场。 |
| 2 | action | `action_record_git_baseline(repo_url=repo_url, target_branch=target_branch)` | 记录业务分支当前 revision，后续用来证明 metadata_branch 模式不会改动业务代码分支。 |
| 3 | action | `action_gui_add_workspace(provider="codeup", repo_url=repo_url, mode="metadata_branch", display_name=display)` | 模拟 GUI 添加 Codeup workspace，并选择 metadata branch 作为 metadata 存储模式。 |
| 4 | assert | `assert_gui_command_cli_equivalent(gui_command="intern.addWorkspace", cli="internctl workspace create --provider codeup --mode metadata_branch")` | 断言 GUI 操作可被 CLI 精确复现，避免 GUI 出现 CLI 无法覆盖的隐藏参数。 |
| 5 | wait | `wait_for_workspace_registered(display_name=display)` | 等待 workspace registry 写入完成，作为后续 metadata branch 检查的前置条件。 |

| 9 | action | `action_cli_workspace_delete(display_name=display, confirm=True)` | 通过 CLI 删除该 workspace，用 CLI 删除路径覆盖 GUI 添加后资源能被统一清理的场景。 |
| 10 | assert | `assert_workspace_removed(display_name=display)` | 断言本机 registry 已移除该 workspace。 |
| 11 | assert | `assert_business_branch_unchanged(repo_url=repo_url, before_rev=before_rev)` | 删除后再次断言业务分支未变化，确保删除流程没有误 push 或 reset 业务分支。 |

## 通过标准

- metadata branch workspace 创建成功且可被 registry/relay 查询。
- 业务 target branch revision 不因 workspace add/delete 改变。
- 删除只清理 workspace 记录和本 case metadata。

## 失败分类

- metadata 写进业务 branch：产品 bug。
- GUI 与 CLI 参数不一致：产品 bug。
- 删除后残留 active workspace record：产品 bug。
