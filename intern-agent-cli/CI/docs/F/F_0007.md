# F_0007 Intern create and status GUI/CLI contract

<!-- METADATA:TYPE=F,ID=F_0007,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=2 -->

## 目标

验证在已存在 workspace 中创建 intern 的 GUI/CLI 合约：创建飞书群、metadata、status、tree projection 和 relay registry 都正确。该用例不发送任务 prompt，不验证 agent 业务能力。

## 资源

- workspace: case 初始化创建 `ci_f_0007_workspace_{run_id}`。
- intern: `intern_ci_f_0007_worker_{run_id}`。

## 剧本

| # | 类型 | 操作 | 注释 |
|---|------|------|------|
| 1 | action | `action_reset_case_namespace(case_id="F_0007")` | 初始化 intern 创建 case 的 namespace，只清理本 case workspace、intern 和群记录。 |
| 2 | action | `action_create_workspace_for_case(provider="local", mode="local_only")` | 创建受控 local workspace，作为创建 intern 的目标 project。 |
| 3 | action | `action_gui_create_intern(workspace=workspace, intern=intern, backend="codex")` | 模拟用户通过 GUI 在该 workspace 下创建 Codex intern。 |
| 4 | assert | `assert_gui_command_cli_equivalent(gui_command="intern.createIntern", cli="internctl create --type codex")` | 断言 GUI 创建 intern 与 CLI `internctl create` 参数等价。 |
| 5 | wait | `wait_for_intern_status(intern=intern, status="Idle")` | 等待 intern 初始化完成并进入 Idle，避免马上读取 metadata 时状态尚未落盘。 |
| 6 | assert | `assert_intern_metadata(project=workspace_display, intern=intern, backend="codex")` | 断言 intern metadata 中 project、intern 名称和 backend 均正确。 |
| 7 | assert | `assert_feishu_group_created(project=workspace_display, intern=intern)` | 断言创建 intern 会创建或绑定对应飞书群。 |
| 8 | assert | `assert_relay_registry_entry(project=workspace_display, intern=intern)` | 断言 relay registry 中存在该 intern，验证 daemon 到 relay 的注册链路。 |
| 9 | assert | `assert_tree_projection_contains(workspace=workspace_display, intern=intern, status="Idle")` | 断言 GUI TreeView 展示与真实 intern status 一致。 |
| 10 | action | `action_cli_intern_status(intern=intern, project=workspace_display, json=True)` | 用 CLI 读取同一 intern 状态，作为 GUI/registry 之外的独立观测入口。 |
| 11 | assert | `assert_status_payload(status="Idle", backend="codex", project=workspace_display)` | 断言 CLI status JSON 返回 Idle、Codex backend 和正确 project。 |

## 通过标准

- GUI create intern 与 CLI create 参数完全等价。
- intern metadata、status、Feishu group、relay registry、TreeView projection 都一致。
- 不启动业务 task，不要求 Codex 做任何修改。

## 失败分类

- 同一 workspace 下创建后 status 缺失或 project 错：产品 bug。
- Feishu group 创建失败但 CLI 返回成功：产品 bug。
- TreeView projection 与 status 文件不一致：产品 bug。
