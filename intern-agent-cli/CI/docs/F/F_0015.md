# F_0015 Question card callback autofill

<!-- METADATA:TYPE=F,ID=F_0015,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=14 -->

## 目标

验证 relay/daemon 的 question card 与 callback 自动回填链路：pending question 注册、卡片可见、CI synthetic callback 路由、callback answer 写回、poll/consume 后 pending 清理、过期 callback 不重建或改写状态。该用例不要求 agent 继续执行。

## 资源

- project: `ci_f_0015_project_{run_id}`。
- intern: `intern_ci_f_0015_worker_{run_id}`。
- question id: case 派生。
- deployed relay: 必须启用 CI synthetic Feishu ingress；`/api/ci/card_callback` 返回 HTTP 403 时分类为部署/环境配置问题，不是 case 实现失败。

## 通过标准

- 注册后只出现一张卡。
- CI synthetic callback 入口在 debug 部署中启用；403 是部署配置失败。
- callback answer 可被 runtime/pending store poll 读取。
- poll 消费 answer 后 pending 清理。
- 重复/不存在 callback 被 relay 作为 transport 事件投递，但 daemon 异步 drop/no-op，不重建 pending、不覆盖答案。

## 失败分类

- 多张卡或无卡：产品 bug。
- callback ok 但 answer 未写回：产品 bug。
- unauthorized user 可回答：产品 bug。
- `/api/ci/card_callback` 返回 403：debug deployment/config issue，需要启用 CI synthetic ingress。
- 重复/不存在 callback 期待同步 HTTP reason：剧本/CI 断言错误；当前产品合约是 daemon 异步 no-op/drop evidence。
