# F_0001 Codeup repo_dotdir workspace add/remove

<!-- METADATA:TYPE=F,ID=F_0001,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=2 -->

## 目标

验证 GUI Add Workspace 与 CLI 对 Codeup `repo_dotdir` workspace 的行为等价：创建、registry 写入、metadata root、relay sync 和删除都可判定。该用例不创建 intern，不启动 agent。

## 资源

- debug machine: `debug-a` 或 `debug-b`。
- repo: `ENTERPRISE_CI_CODEUP_NONPROTECTED_REPO` 或 CI 默认 nonprotected Codeup repo。
- workspace display: `ci_f_0001_codeup_dotdir_{run_id}`。
- mode: `repo_dotdir`。

## 剧本

| # | 类型 | 操作 | 注释 |
|---|------|------|------|
| 1 | action | `action_reset_case_namespace(case_id="F_0001")` | 初始化本 case namespace，只清理 `F_0001` 前缀资源，避免影响主管或其他 case 的现场。 |
| 2 | action | `action_gui_add_workspace(provider="codeup", repo_url=repo_url, mode="repo_dotdir", display_name=display)` | 模拟 GUI 添加 Codeup workspace，选择 `repo_dotdir` 模式并写入本次运行专属 display name。 |
| 3 | assert | `assert_gui_command_cli_equivalent(gui_command="intern.addWorkspace", cli="internctl workspace create --provider codeup --mode repo_dotdir")` | 断言 GUI handler 精确映射到 CLI 创建命令；如果无法等价表达，应判定为 GUI/CLI contract bug。 |
| 4 | wait | `wait_for_workspace_registered(display_name=display)` | 等待 workspace 出现在本机 registry，消除异步写入造成的时序误判。 |
| 5 | assert | `assert_workspace_record(provider="codeup", mode="repo_dotdir", repo_url=repo_url, display_name=display)` | 断言 registry 中 provider、mode、repo_url、display name 与用户输入一致。 |
| 6 | assert | `assert_workspace_metadata_root(kind="repo_dotdir", path_under_repo=True)` | 断言 `repo_dotdir` 模式把 metadata 放在仓库内约定目录，而不是误写到 metadata branch 或全局目录。 |
| 7 | assert | `assert_relay_workspace_sync(display_name=display, provider="codeup")` | 断言 relay 能看到该 workspace，验证 daemon 到 relay 的 workspace 同步链路。 |
| 8 | action | `action_gui_remove_workspace(display_name=display)` | 模拟 GUI 删除刚创建的 workspace，覆盖用户从界面移除 workspace 的路径。 |
| 9 | wait | `wait_for_workspace_removed(display_name=display)` | 等待本地删除动作完成并从 registry 消失，避免马上断言导致误报。 |
| 10 | assert | `assert_workspace_state_removed(display_name=display, local_registry=True, relay_registry=True)` | 断言本机 registry 和 relay registry 都已删除该 workspace，且没有误删其他 namespace 资源。 |

## 通过标准

- GUI add/remove 和 CLI contract 完全等价。
- workspace 创建后 registry、metadata root、relay sync 都一致。
- 删除后本 case workspace 不再出现在本机 registry 和 relay registry。

## 失败分类

- GUI 无法映射到 CLI：产品 bug。
- 创建成功但 registry/relay 不一致：产品 bug。
- 删除误删非本 case workspace：CI 逻辑错误或产品 bug，按 evidence 分类。
