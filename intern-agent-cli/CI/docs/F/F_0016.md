# F_0016 Slash config mode persistence

<!-- METADATA:TYPE=F,ID=F_0016,STATUS=ApprovedForImplementation,OWNER=intern_ci_lead,SESSION=62 -->

## 目标

验证飞书 slash `/config` 卡片对 trigger mode、detail mode 和 no-collapse mode 的保存与读取。该用例只测配置和卡片 callback，不要求 agent 响应业务 prompt。

- intern: `intern_ci_f_0016_worker_{run_id}`。

## 通过标准

- `/config` 能打开卡片。
- `/config` 卡片必须有 Save 和 Cancel，Cancel 不产生配置副作用。
- trigger/detail/no-collapse mode 保存后可再次读取。
- daemon-local detail/no-collapse mode 与 relay-local trigger mode 均一致。

## 失败分类

- detailed mode 未保存或只保存在卡片本地状态：产品 bug。
- no-collapse mode 未保存或只保存在卡片本地状态：产品 bug。
- card callback 成功但 registry 未更新：产品 bug。
- slash 被当作普通 prompt 注入 agent：产品 bug。
- 卡片缺少 Cancel 或 Cancel 仍写入配置：产品 bug。
