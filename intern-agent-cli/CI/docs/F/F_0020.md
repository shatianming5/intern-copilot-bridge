# F_0020 Slash routing errors, RBAC, unknown fallback

<!-- METADATA:TYPE=F,ID=F_0020,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=2 -->

## 目标

验证 slash 命令的错误和权限边界：未知命令给 help fallback，非 owner 不能执行敏感 callback，未映射群不能 spam，错误 project/intern 有结构化提示。

## 资源

- owner user and non-owner user。
- mapped intern chat: `ci_f_0020_project_{run_id}`。
- unmapped test chat。

## 剧本

| # | 类型 | 操作 | 注释 |
|---|------|------|------|
| 1 | action | `action_reset_case_namespace(case_id="F_0020")` | 初始化 slash 错误和权限 case 的 namespace，只清理本 case group、config 和错误消息记录。 |
| 2 | action | `action_seed_group(project=project, intern=intern, owner=owner_user)` | 构造已映射 chat，并指定 owner，用于区分 owner 和非 owner 权限。 |

| 13 | assert | `assert_slash_error_reply(reason_kind="project_not_found", structured=True)` | 断言 project 不存在时返回结构化错误，便于 GUI/飞书侧展示和 CI 分类。 |

## 通过标准

- unknown command 有帮助提示，不进入 agent prompt。
- unmapped chat 不刷屏，不创建资源。
- 非 owner 敏感 callback 被拒绝，配置不变。
- 错误 project/intern 返回结构化错误。

## 失败分类

- 非 owner 修改配置或启动 helper：产品 bug。
- unknown slash 被注入 agent：产品 bug。
- unmapped chat 创建 registry 或群：产品 bug。
