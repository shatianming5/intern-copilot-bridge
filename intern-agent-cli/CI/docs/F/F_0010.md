# F_0010 Intern delete and force delete guard

<!-- METADATA:TYPE=F,ID=F_0010,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=2 -->

## 目标

验证 intern delete 的安全合约：Idle intern 普通删除应清理完整；Working intern 普通删除必须拒绝并保留现场；force delete 才能清理 runtime、session registry、task metadata 和 group registry。

## 资源

- workspace: `ci_f_0010_workspace_{run_id}`。
- idle intern: `intern_ci_f_0010_idle_{run_id}`。
- working intern: `intern_ci_f_0010_working_{run_id}`。

## 剧本

| # | 类型 | 操作 | 注释 |
|---|------|------|------|
| 1 | action | `action_reset_case_namespace(case_id="F_0010")` | 初始化删除保护 case 的 namespace，只清理本 case 的 workspace、intern、session 和 task。 |
| 2 | action | `action_create_workspace_for_case(provider="local", mode="local_only")` | 创建受控 workspace，承载 idle 和 working 两类 intern。 |

## 通过标准

- 普通删除只允许安全状态。
- Working 删除拒绝后现场完整保留。
- force delete 清理本 case intern 的全部状态，不影响 workspace。

## 失败分类

- 普通删除 Working intern 成功：产品 bug。
- force delete 留下 session/task/runtime 残留：产品 bug。
- 删除 workspace 或其他 intern：产品 bug 或 CI 逻辑错误。
