# F_0013 Relay chat create/delete project scope

<!-- METADATA:TYPE=F,ID=F_0013,STATUS=ReviewDraft,OWNER=intern_ci_lead,SESSION=2 -->

## 目标

验证 relay `/api/chat/create` 与 `/api/chat/delete` 的 project-scoped lifecycle：同名 intern 跨 workspace 不串群，重复 create 复用，delete 只删除指定 project。

## 资源

- project A: `ci_f_0013_project_a_{run_id}`。
- project B: `ci_f_0013_project_b_{run_id}`。
- intern name: `intern_ci_f_0013_same_name`。

## 剧本

| # | 类型 | 操作 | 注释 |
|---|------|------|------|
| 1 | action | `action_reset_case_namespace(case_id="F_0013")` | 初始化 relay chat case 的 namespace，只清理本 case 的 project/chat 绑定。 |
| 2 | action | `action_relay_post("/api/chat/create", {"project": project_a, "intern_name": intern})` | 直接调用 relay 创建 project A 下的 intern chat，验证 relay API 本身。 |
| 3 | assert | `assert_chat_created(project=project_a, intern=intern, chat_id_present=True)` | 断言 project A 的 chat 创建成功并返回 chat_id。 |
| 4 | action | `action_relay_post("/api/chat/create", {"project": project_a, "intern_name": intern})` | 对同一 project/intern 再次创建 chat，覆盖幂等创建行为。 |
| 5 | assert | `assert_chat_reused(project=project_a, intern=intern, same_chat_id=True)` | 断言重复 create 复用原 chat_id，不创建重复群。 |
| 6 | action | `action_relay_post("/api/chat/create", {"project": project_b, "intern_name": intern})` | 在 project B 下创建同名 intern chat，覆盖跨 project 同名 intern。 |
| 7 | assert | `assert_chat_created(project=project_b, intern=intern, chat_id_present=True)` | 断言 project B 的 chat 独立创建成功。 |
| 8 | assert | `assert_chat_ids_distinct(project_a, project_b, intern)` | 断言 project A 和 project B 的同名 intern 使用不同 chat_id，避免跨 project 串群。 |
| 9 | action | `action_relay_post("/api/chat/delete", {"project": project_a, "intern_name": intern})` | 删除 project A 的 chat，验证 delete 只作用于指定 project scope。 |
| 10 | assert | `assert_chat_removed(project=project_a, intern=intern)` | 断言 project A 的 chat 绑定已删除。 |
| 11 | assert | `assert_chat_still_exists(project=project_b, intern=intern)` | 断言删除 project A 不会影响 project B 的同名 intern chat。 |
| 12 | action | `action_relay_post("/api/chat/delete", {"intern_name": intern})` | 发送缺少 project 的 delete 请求，覆盖 relay 参数校验。 |
| 13 | assert | `assert_request_failed(reason_kind="missing_project")` | 断言缺少 project 的 delete 请求失败，避免误删所有同名 intern chat。 |

## 通过标准

- project A/B 同名 intern 使用不同 chat entry。
- 重复 create 对同 project 幂等复用。
- delete 只影响指定 project。

## 失败分类

- 同名跨 project 串群：产品 bug。
- delete 无 project 仍执行：产品 bug。
- Feishu API 外部失败：environment/external，保留 request/response。
